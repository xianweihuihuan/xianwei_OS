# xianwei_OS

`xianwei_OS` 是一个基于《操作系统真相还原》设计思路实现的 32 位 x86 小型操作系统项目，目标是在尽量贴近底层硬件的前提下，把一个可启动、可交互、可运行用户程序的内核逐步搭建起来。

这个项目不仅覆盖了从 `MBR`、`Loader` 到内核加载的启动链路，也实现了中断管理、内存分页、线程调度、系统调用、用户进程、文件系统和 Shell 等核心组件。相比只停留在“能点亮屏幕”或“能打印字符串”的练习型代码，它更接近一个具备完整 OS 主干的实验系统。

## 项目特性

- 从 `MBR` 到 `Loader` 再到内核入口，完整走通启动流程
- 支持从实模式切换到 32 位保护模式，并建立分页机制
- 基于 `IDT`、时钟中断、键盘中断完成基础中断与设备管理
- 实现内核线程、调度器、同步原语与基本进程管理
- 提供用户态支持，包含 `fork / exec / wait / exit` 等关键机制
- 提供系统调用层，连接用户程序与内核服务
- 基于磁盘镜像实现简化文件系统，支持目录、普通文件与路径解析
- 内置 Shell，支持常用命令、外部程序加载与管道重定向

## 已实现能力

### 1. 启动与内核装载

项目包含独立的引导扇区与加载器代码：

- `boot/MBR.s` 负责最初阶段的引导与磁盘读取
- `boot/loader.s` 负责内存探测、进入保护模式、建立页表并装载内核
- `kernel.bin` 以 ELF 形式链接，由加载器解析段信息后搬运到内存执行

这部分让整个系统具备了较完整的“上电 -> 引导 -> 进入内核”的链路，而不是依赖宿主程序直接跳转。

### 2. 中断、内存与线程系统

内核初始化过程中完成了以下基础设施构建：

- 中断描述符表初始化
- 内存池与分页机制初始化
- 线程子系统初始化
- 时钟中断驱动的调度支持
- 键盘输入与控制台输出支持
- `TSS` 初始化，为用户态切换提供基础

这些模块共同构成了内核最核心的运行时环境。

### 3. 用户进程与系统调用

项目已经具备用户程序运行能力，并提供了较完整的内核接口：

- 进程创建：`fork`
- 程序替换：`execv`
- 进程回收：`wait / exit`
- 文件读写：`open / close / read / write / lseek`
- 目录操作：`mkdir / rmdir / opendir / readdir / chdir / getcwd / stat`
- 管道通信：`pipe` 与文件描述符重定向

这意味着系统已经从“只有内核线程”演进到“可以运行用户态程序并通过系统调用访问内核资源”。

### 4. 文件系统与交互式 Shell

项目在磁盘镜像上实现了简化文件系统，支持：

- 分区格式化与挂载
- 超级块、inode、位图管理
- 目录项增删查找
- 普通文件创建、打开、删除与读写
- 路径解析与当前工作目录切换

在此基础上，系统提供了可交互的 Shell，支持以下内建命令：

- `ls`
- `cd`
- `pwd`
- `ps`
- `clear`
- `mkdir`
- `rmdir`
- `rm`
- `help`

同时也支持加载外部用户程序。当前仓库中的 `command/cat.c`、`command/prog_pipe.c` 展示了用户程序、管道与进程协作的基本能力。

## 项目结构

```text
xianwei_OS/
├── boot/        # MBR、Loader 与引导相关代码
├── kernel/      # 内核初始化、中断、内存管理等核心模块
├── thread/      # 线程、调度与同步机制
├── userprog/    # 用户进程、fork/exec、TSS、系统调用入口
├── fs/          # 文件系统、inode、目录、文件操作
├── device/      # IDE 磁盘、键盘、定时器、控制台、IO 队列
├── shell/       # Shell、内建命令、管道支持
├── command/     # 用户态示例程序
├── lib/         # 内核态 / 用户态公共基础库
├── test/        # 一些实验与验证代码
├── bochsrc.disk # Bochs 运行配置
└── CMakeLists.txt
```

## 构建与运行

### 环境依赖

建议在 Linux 环境下准备以下工具：

- `gcc`，并具备 32 位编译支持
- `ld`
- `nasm`
- `cmake`
- `bochs`

### 构建内核

当前仓库使用 `CMake` 生成 `kernel.bin`：

```bash
mkdir -p build
cd build
cmake ..
cmake --build .
```

### 运行方式

仓库中已提供 `Bochs` 配置文件与磁盘镜像，可结合本地环境直接启动：

```bash
bochs -f bochsrc.disk
```

如果需要完整复现实验过程，可以在此基础上继续结合引导程序、磁盘镜像写入流程和用户程序装载脚本进行调试与扩展。

## 设计重点

`xianwei_OS` 的重点并不在于追求“大而全”，而是在尽可能小的系统规模下，把操作系统中的关键机制真正串起来：

- 启动链路是可追踪的
- 内核与用户态边界是清晰的
- 调度、内存、文件、进程和 Shell 之间能够形成闭环
- 代码以模块拆分为主，适合继续扩展为更完整的教学型 OS

这个项目以 32 位 x86 平台为实验对象，强调从底层出发理解操作系统的核心机制。对于希望系统性学习引导加载、内核设计、进程管理、文件系统和用户态执行链路的人来说，它是一个非常适合持续迭代的实践型项目。
